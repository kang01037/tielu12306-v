基于Spring Cloud微服务架构的铁路12306售票系统，模拟中国铁路互联网售票系统核心功能。

## 项目介绍

本项目实现火车票查询、预订、支付、退改签等核心功能，重点解决高并发场景下的库存扣减、订单一致性等关键技术难题。

## 微服务通信

本系统使用 **OpenFeign** 实现服务间调用：

```
service-order (订单服务)
    │
    ├── Feign调用 ──→ service-user (获取乘车人信息)
    │
    ├── Feign调用 ──→ service-train (查询车次、车站信息)
    │
    └── Feign调用 ──→ service-ticket (锁定/解锁座位、释放库存)
```

## 快速开始

### 环境要求

- JDK 21+
- Maven 3.9+
- MySQL 8.0+
- Redis 7.x+
- Nacos 2.x (服务注册与发现)

### 1. 启动Nacos

确保Nacos已启动并运行在 `localhost:8848`

```bash
# 下载并启动Nacos
cd nacos/bin
./startup.sh -m standalone  # Linux/Mac
startup.cmd -m standalone    # Windows
```

### 2. 初始化数据库

```bash
# 执行SQL脚本创建数据库和表
mysql -u root -p < sql/init.sql
```

### 3. 配置Redis

确保Redis服务运行在 `localhost:6379`

### 4. 编译项目

```bash
mvn clean install -DskipTests
```

### 5. 启动服务

**重要**: 必须先启动Nacos，然后按以下顺序启动服务：

```bash
# 1. 启动用户服务
cd service/service-user
mvn spring-boot:run

# 2. 启动列车服务
cd service/service-train
mvn spring-boot:run

# 3. 启动余票服务
cd service/service-ticket
mvn spring-boot:run

# 4. 启动订单服务 (依赖其他服务，通过Feign调用)
cd service/service-order
mvn spring-boot:run

# 5. 启动支付服务
cd service/service-pay
mvn spring-boot:run

# 6. 启动网关
cd gateway
mvn spring-boot:run
```

### 5. 验证服务

访问车站列表接口：
```bash
curl http://localhost:8080/api/station
```

## 核心流程

### 查票流程

```
用户输入 出发站/到达站/日期
        │
        ▼
┌───────────────────┐
│  Redis 缓存查询    │───── 命中 ────▶ 直接返回结果
└───────┬───────────┘
        │ 未命中
        ▼
┌───────────────────┐
│  MySQL 查询车次    │
│  及余票信息        │
└───────┬───────────┘
        │
        ▼
      返回结果
```

### 下单流程

```
用户提交订单 (车次/日期/乘车人/席位)
        │
        ▼
┌───────────────────┐
│  Redis 分布式锁    │ ── 获取失败 ──▶ 返回繁忙
└───────┬───────────┘
        │ 获取锁
        ▼
┌───────────────────┐
│  扣减库存          │ ── 库存不足 ──▶ 释放锁, 返回无票
│  (乐观锁 UPDATE)   │
└───────┬───────────┘
        │ 扣减成功
        ▼
┌───────────────────┐
│  生成订单          │
│  (状态: 待支付)    │
└───────┬───────────┘
        │
        ▼
┌───────────────────┐
│  发送延时消息       │ ── 30分钟后检查支付状态
│  (RabbitMQ)       │     未支付则自动取消
└───────────────────┘
```

## 高并发方案

### 查票高并发
- 多级缓存：本地Caffeine → Redis → MySQL
- 读写分离：MySQL主从复制

### 下单高并发
- 分布式锁：Redis Redisson锁，车次+日期+席位粒度
- 乐观锁：库存表version字段，防止超卖
- 异步化：RabbitMQ异步处理订单后置逻辑

### 数据一致性
- 库存一致性：Redis预扣 + MySQL乐观锁
- 订单一致性：本地事务 + 消息队列
